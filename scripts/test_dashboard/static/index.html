LLM Connector — Test Dashboard

Тест вызывает реальный complete(): пишет в llm_request_logs и при ошибках может увеличить failure_count (как в проде). LLM_DEPLOYMENT_CODE=test_dashboard на время теста.

Каждое обновление заново читает последние N записей из БД (сверху — самые новые). Фильтр «Скрипт» ограничивает выборку: при выбранном скрипте записи других пайплайнов не показываются.
